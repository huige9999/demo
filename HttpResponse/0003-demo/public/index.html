SSE vs 普通流式响应

同一段 token 序列，两种写法、两种消费方式。底层都是字节流，差别在协议与 API。

文档：docs/0003-SSE响应和普通流响应.md

SSE · EventSource

const es = new EventSource('/api/sse');
es.onmessage = (e) => {
  const { token } = JSON.parse(e.data);
  output.textContent += token;
};
es.addEventListener('done', () => es.close());

事件日志

普通流 · fetch + ReadableStream

const res = await fetch('/api/stream');
const dec = new TextDecoder();
let text = '';
for await (const chunk of res.body) {
  text += dec.decode(chunk, { stream: true });
  output.textContent = text;
}

产出

chunk 日志

观察要点

  • SSE：日志里每条都是完整的 data: {...} 消息；浏览器按 \n\n 切帧。
  • 普通流：日志里是 TCP chunk，可能一次含多个 token，也可能一个 token 拆成多块（本 demo 每次 write 一个字符，通常一块 = 一个 token）。
  • 两者 UI 最终文本相同，但触发 UI 更新的粒度不同：SSE 按「事件」，普通流按「chunk」。
